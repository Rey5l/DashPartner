# Интеграция CryptoBot

## Описание

Полная интеграция с CryptoBot API для пополнения и вывода средств в USDT с системой заявок и админ-панелью.

## Установленные компоненты

### 1. Сервис CryptoBot (`services/crypto_service.py`)
- `create_invoice()` - создание инвойса для пополнения
- `check_invoice()` - проверка статуса инвойса
- `create_check()` - создание чека для вывода
- `get_balance()` - получение баланса кошелька

### 2. База данных (`services/database_service.py`)

#### Таблица `topup_transactions` (пополнения):
- `id` - ID транзакции
- `user_id` - ID пользователя
- `invoice_id` - ID инвойса CryptoBot
- `amount` - сумма в USDT
- `status` - статус (pending/paid)
- `created_at` - дата создания
- `paid_at` - дата оплаты

#### Таблица `withdrawal_requests` (заявки на вывод):
- `id` - ID заявки
- `user_id` - ID пользователя
- `amount` - сумма в USDT
- `status` - статус (pending/approved/refunded/declined)
- `created_at` - дата создания
- `processed_at` - дата обработки
- `processed_by` - ID админа, обработавшего заявку
- `check_url` - ссылка на чек (для одобренных)
- `decline_reason` - причина отклонения

#### Методы для пополнений:
- `create_topup_transaction()` - создать транзакцию
- `get_topup_transaction()` - получить транзакцию
- `mark_topup_paid()` - отметить как оплаченную и начислить баланс
- `get_user_topup_transactions()` - история пополнений

#### Методы для заявок на вывод:
- `create_withdrawal_request()` - создать заявку
- `get_withdrawal_request()` - получить заявку
- `get_withdrawals_by_status()` - получить заявки по статусу
- `count_withdrawals_by_status()` - подсчитать заявки по статусу
- `approve_withdrawal()` - одобрить заявку
- `decline_withdrawal()` - отклонить заявку (с возвратом или без)
- `get_user_withdrawal_requests()` - история заявок пользователя

### 3. Обработчики для пользователей (`routes/balance_routes.py`)
- **Пополнение баланса** через CryptoBot
- **Проверка оплаты** инвойса
- **Создание заявки на вывод** (вместо немедленного вывода)
- **Уведомление администраторов** о новых заявках

### 4. Админ-панель (`routes/admin_routes.py`)

#### Баланс резерва:
- Просмотр баланса кошелька CryptoBot
- Статус резерва (критически низкий / низкий / средний / достаточный)
- Пополнение резерва через инвойс
- Уведомление других админов о пополнении

#### Заявки на вывод:
- Просмотр заявок по статусам:
  - 🕓 Ожидающие (pending)
  - ✅ Подтвержденные (approved)
  - ↩️ Отклоненные с возвратом (refunded)
  - ❌ Отклоненные без возврата (declined)
- Детальный просмотр каждой заявки
- Действия с заявками:
  - ✅ Подтвердить - создает чек и отправляет пользователю
  - ❌ Отклонить без возврата - средства не возвращаются
  - ↩️ Отклонить с возвратом - средства возвращаются на баланс

### 5. Клавиатуры (`keyboards/reply_keyboard.py`)
Добавлены кнопки в админскую клавиатуру:
- 💰 Баланс резерва
- 🧾 Заявки на вывод

## Настройка

1. Получите токен CryptoBot API на https://t.me/CryptoBot
2. Добавьте токен в `.env`:
   ```
   CRYPTO_BOT_TOKEN=your_token_here
   ```
3. Убедитесь, что в `config.py` указаны ID администраторов в `ADMIN_IDS`

## Использование

### Для пользователей:

#### Пополнение баланса
1. Нажать кнопку "Пополнить" в профиле (callback_data: `topup`)
2. Ввести сумму в USDT (минимум 1 USDT)
3. Получить ссылку на оплату через CryptoBot
4. После оплаты нажать "Проверить оплату"
5. Средства автоматически зачисляются на баланс

#### Вывод средств
1. Нажать кнопку "Вывод" в профиле (callback_data: `withdraw`)
2. Ввести сумму для вывода (минимум 1 USDT)
3. Заявка создается и отправляется администраторам
4. Средства списываются с баланса
5. После одобрения админом пользователь получает чек CryptoBot

### Для администраторов:

#### Баланс резерва
1. Нажать "💰 Баланс резерва" в админской клавиатуре
2. Просмотреть текущий баланс и статус
3. При необходимости пополнить резерв через инвойс

#### Обработка заявок на вывод
1. Нажать "🧾 Заявки на вывод"
2. Выбрать категорию (ожидающие, подтвержденные и т.д.)
3. Открыть конкретную заявку
4. Выбрать действие:
   - **Подтвердить** - создается чек, отправляется пользователю
   - **Отклонить без возврата** - средства не возвращаются
   - **Отклонить с возвратом** - средства возвращаются на баланс

## Workflow вывода средств

1. **Пользователь создает заявку**
   - Средства списываются с баланса
   - Создается запись в `withdrawal_requests` со статусом `pending`
   - Все админы получают уведомление

2. **Админ обрабатывает заявку**
   - **Одобрение**: создается чек CryptoBot, отправляется пользователю
   - **Отклонение без возврата**: средства остаются списанными
   - **Отклонение с возвратом**: средства возвращаются на баланс

3. **Пользователь получает уведомление**
   - При одобрении: ссылка на чек
   - При отклонении: информация о возврате или его отсутствии

## Статусы резерва

- 🔴 **КРИТИЧЕСКИ НИЗКИЙ** - меньше 1 USDT
- 🟡 **НИЗКИЙ** - от 1 до 10 USDT
- 🟢 **СРЕДНИЙ** - от 10 до 50 USDT
- 💚 **ДОСТАТОЧНЫЙ** - больше 50 USDT

## API Endpoints CryptoBot

- `POST /api/createInvoice` - создание инвойса
- `GET /api/getInvoices` - получение информации об инвойсах
- `POST /api/createCheck` - создание чека
- `GET /api/getBalance` - получение баланса

## Безопасность

- Все транзакции сохраняются в базе данных
- Проверка на дублирование платежей
- Проверка баланса перед созданием заявки на вывод
- Токен CryptoBot хранится в переменных окружения
- Только администраторы могут обрабатывать заявки
- Уведомления всех админов о новых заявках

## Тестирование

1. Убедитесь, что токен CryptoBot установлен в `.env`
2. Убедитесь, что ваш ID добавлен в `ADMIN_IDS` в `config.py`
3. Запустите бота
4. Попробуйте пополнить баланс минимальной суммой
5. Проверьте, что средства зачислились
6. Попробуйте создать заявку на вывод
7. Проверьте, что заявка появилась в админ-панели
8. Одобрите заявку и проверьте, что чек создан
