# Добавленный функционал админки

## Что было добавлено

### 1. Баланс резерва (💰 Баланс резерва)

**Функционал:**
- Просмотр текущего баланса кошелька CryptoBot в USDT
- Автоматическое определение статуса резерва:
  - 🔴 КРИТИЧЕСКИ НИЗКИЙ (< 1 USDT)
  - 🟡 НИЗКИЙ (1-10 USDT)
  - 🟢 СРЕДНИЙ (10-50 USDT)
  - 💚 ДОСТАТОЧНЫЙ (> 50 USDT)
- Пополнение резерва через инвойс CryptoBot
- Уведомление других администраторов о пополнении

**Как использовать:**
1. Нажмите "💰 Баланс резерва" в админской клавиатуре
2. Просмотрите текущий баланс и статус
3. Нажмите "💵 Пополнить резерв"
4. Введите сумму в USDT
5. Оплатите инвойс через CryptoBot

### 2. Заявки на вывод (🧾 Заявки на вывод)

**Функционал:**
- Просмотр всех заявок на вывод по категориям:
  - 🕓 Ожидающие - требуют обработки
  - ✅ Подтвержденные - чеки созданы и отправлены
  - ↩️ Отклоненные с возвратом - средства возвращены
  - ❌ Отклоненные без возврата - средства не возвращены
- Детальная информация о каждой заявке
- Отображение баланса резерва при просмотре заявки
- Три варианта обработки заявки

**Как использовать:**
1. Нажмите "🧾 Заявки на вывод" в админской клавиатуре
2. Выберите категорию заявок
3. Нажмите на заявку для просмотра деталей
4. Выберите действие:
   - **✅ Подтвердить** - создается чек CryptoBot, отправляется пользователю
   - **❌ Отклонить (без возврата)** - средства остаются списанными
   - **↩️ Отклонить (с возвратом)** - средства возвращаются на баланс пользователя

### 3. Изменения в процессе вывода для пользователей

**Старая логика:**
- Пользователь запрашивал вывод
- Чек создавался немедленно
- Средства списывались сразу

**Новая логика:**
- Пользователь создает заявку на вывод
- Средства списываются с баланса
- Заявка отправляется администраторам
- Администратор обрабатывает заявку
- При одобрении создается чек и отправляется пользователю
- При отклонении средства могут быть возвращены или нет

## Преимущества новой системы

1. **Контроль над выводами** - администраторы проверяют каждую заявку
2. **Мониторинг резерва** - видно, достаточно ли средств для выплат
3. **Защита от мошенничества** - возможность отклонить подозрительные заявки
4. **Гибкость** - можно вернуть средства или оставить списанными
5. **Прозрачность** - все заявки сохраняются в базе данных
6. **Уведомления** - все админы получают уведомления о новых заявках

## Уведомления

### Для администраторов:
- Новая заявка на вывод (с информацией о пользователе и резерве)
- Пополнение резерва другим админом

### Для пользователей:
- Заявка создана (с ID заявки)
- Заявка одобрена (со ссылкой на чек)
- Заявка отклонена (с информацией о возврате)

## Технические детали

### Таблицы базы данных:
- `topup_transactions` - история пополнений
- `withdrawal_requests` - заявки на вывод

### Статусы заявок:
- `pending` - ожидает обработки
- `approved` - одобрена, чек создан
- `refunded` - отклонена с возвратом средств
- `declined` - отклонена без возврата

### Файлы:
- `routes/admin_routes.py` - обработчики админки
- `routes/balance_routes.py` - обработчики для пользователей
- `services/database_service.py` - методы работы с БД
- `services/crypto_service.py` - интеграция с CryptoBot
- `keyboards/reply_keyboard.py` - клавиатуры

## Безопасность

- Только пользователи из `ADMIN_IDS` могут обрабатывать заявки
- Все действия логируются в базе данных
- Сохраняется информация о том, кто обработал заявку
- Токен CryptoBot хранится в переменных окружения
